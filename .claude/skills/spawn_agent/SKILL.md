description: |
  创建独立子代理执行子任务。以下场景【必须】使用，禁止串行用bash代替：
  1. 【多股票/多公司分析】用户要求分析≥2只股票或≥2家公司时，必须为每个对象启动一个子代理并行执行（如"分析茅台和五粮液"→启动2个子代理）
  2. 【单股深度分析】对单只股票做技术面+资金面+基本面完整分析（≥5个工具调用），启动1个stock_expert子代理
  3. 【深度研究隔离】各子任务上下文相互干扰，或单任务≥8轮工具调用
  4. 规划+执行分离（父代理规划，子代理执行）
  不适用：简单单一查询（查股价、查新闻）、工具调用<3次。
  重要：软件开发、代码编写、系统部署、续接开发（"继续开发"/"不要停"/"直到完成"）等任务，调用时必须传 background=true，否则会阻塞主流程。

## 调用方式

spawn_agent 由框架内部调用，无需通过 bash 执行脚本。直接作为工具调用即可：

```json
{
  "task": "完整的子任务描述，包含所有必要背景信息",
  "role": "stock_expert"
}
```

## 参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task | string | 是 | 子任务完整描述，必须包含所有必要背景信息（子代理看不到父代理对话历史） |
| role | string | 否 | 专家角色: stock_expert/military_expert/political_expert/general_expert/software_expert，默认 "general_expert" |
| background | boolean | 否 | true=后台异步执行（立即返回 task_id，子代理在后台独立运行，最长5小时，迭代不限，完成后自动通知用户）。**预估超过15分钟的任务必须设为true**。默认 false |

## spawn_agent

### 三类必须使用场景

**场景一：多目标并发独立分析**

用户要求同时分析多个对象（多家公司、多个地区、多个产品），且各对象分析逻辑完全独立时，为每个对象各启动一个子代理并行执行，最后由你汇总对比。

示例：
- "同时分析茅台、五粮液、泸州老窖三家公司的财务状况并对比"
  → 并行启动 3 个子代理，各自独立分析一家公司，你负责最终对比汇总

**场景二：需要上下文隔离的深度研究**

各子任务之间上下文相互干扰、或单个子任务需要超过 5 轮工具调用时，用子代理隔离各自的对话上下文，避免信息污染。

示例：
- "帮我做竞品分析，每家公司的分析逻辑完全独立"
  → 为每家公司启动独立子代理，各自有完整的工具调用循环，互不干扰

**场景三：规划 + 执行分离**

复杂任务需要先规划再执行，且执行阶段各模块互相独立时，由你负责规划和最终汇总，将各执行模块委托给子代理。

示例：
- 父代理：制定研究大纲、分配子任务 → 子代理 A：数据收集 → 子代理 B：竞品分析 → 父代理：整合输出

### 使用规范

- `task` 参数必须包含完整的子任务描述和所有必要背景信息，子代理看不到父代理的对话历史
- `role` 参数选择与任务匹配的专家角色，让子代理具备对应领域的分析框架和行为准则
- 子代理默认拥有完整工具集，无需限制
- 子代理完成后将结果返回给你，由你负责最终输出

### role 参数说明

| role 值 | 适用场景 |
|---------|---------|
| `stock_expert` | 股票技术面/资金面/基本面分析、预测、操作建议 |
| `military_expert` | 武装冲突分析、军事力量对比、战场态势评估 |
| `political_expert` | 地缘政治、外交动态、制裁贸易、认知战分析 |
| `general_expert` | 通用研究、信息收集、报告整理（默认值） |
| `software_expert` | 代码开发、架构设计、调试、技术方案 |

## 调用示例

**股票分析（并行多个子代理）：**
```json
{
  "task": "分析贵州茅台600519.SH的财务状况，包括营收、净利润、ROE，输出结构化摘要",
  "role": "stock_expert"
}
```

**军事情报分析：**
```json
{
  "task": "分析当前俄乌冲突最新态势，包括战线变化、双方兵力损耗和近期战略动向",
  "role": "military_expert"
}
```

**地缘政治研究：**
```json
{
  "task": "分析中美关税博弈最新进展及对全球供应链的影响",
  "role": "political_expert"
}
```

**软件开发任务（必须 background=true）：**
```json
{
  "task": "用 Python 实现一个异步任务队列，支持优先级、重试机制和超时控制，提供完整可运行代码。完成后发送验收报告。",
  "role": "software_expert",
  "background": true
}
```

**通用研究（默认）：**
```json
{
  "task": "收集并整理2025年全球新能源汽车市场份额数据，输出各主要厂商排名报告",
  "role": "general_expert"
}
```

**后台异步开发任务（background=true）：**
```json
{
  "task": "按以下设计方案开发用户中心系统：\n\n技术栈：FastAPI + SQLite + Vue 3\n\n功能：用户注册登录、个人信息管理、权限控制\n\n要求：所有代码在 docker_operator toolbox 中开发，配置化，禁止硬编码。完成后发送验收报告。",
  "role": "software_expert",
  "background": true
}
```
说明：background=true 时父代理立即返回，子代理在后台独立运行（最长 5 小时，迭代次数不限），完成后系统自动通知用户。

**必须使用 background=true 的场景：**
- 小游戏/小程序完整开发（策划→写代码→测试→上传，通常 30-60 分钟）
- 软件项目开发（多文件开发+测试+部署）
- 大规模数据处理/ETL 任务
- 需要多次迭代修复 bug 的开发任务
- 视频/音频生成等耗时任务
- **续接开发任务**：用户说"继续开发"、"继续做"、"不要停"、"直到完成"等，只要涉及软件开发，必须 background=true

**判断标准：预估执行时间超过 15 分钟的任务，必须用 background=true。**
