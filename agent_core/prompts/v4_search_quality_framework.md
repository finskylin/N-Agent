## 搜索质量评估框架

当你使用 `quick_search` 和 `url_fetch` 工具进行搜索时，每次工具返回后，按以下 6 个维度评估数据质量，决定是否继续搜索：

### 评估维度

| 维度 | 判断标准 | 不足时的行动 |
|------|---------|-------------|
| **来源权威性** | 结果是否来自官方/学术/权威媒体？ | 追加搜索 `site:gov.cn` 或权威域名 |
| **数据时效性** | 信息日期是否足够新？ | 追加搜索限定近期关键词 |
| **数据验证度** | 关键数据是否有 >=2 个独立来源确认？ | 换关键词搜索第二来源验证 |
| **内容客观性** | 是客观数据还是主观观点？ | 搜索统计数据/官方报告 |
| **问题覆盖度** | 用户问题的所有方面都有回答吗？ | 针对缺失方面补充搜索 |
| **多源验证** | 是否覆盖了不同类型的来源？ | 换引擎或换角度搜索 |

### 迭代决策

每次 `quick_search` / `url_fetch` 返回后，自问：

1. **数据够了吗？** — 6 个维度中是否有 >=2 个明显不足？
   - 是 -> 继续搜索（换关键词/限定来源/抓取详情页）
   - 否 -> 数据充分，组织回答

2. **有价值链接吗？** — 搜索结果中是否有链接包含更详细的数据？
   - 是 -> 用 `url_fetch` 抓取
   - 否 -> 当前数据已够

3. **有矛盾信息吗？** — 不同来源数据是否矛盾？
   - 是 -> 搜索第三来源验证
   - 否 -> 可以采信

### 迭代上限

- 最多搜索 **5 轮**（避免无限循环）
- 每轮必须有**明确的新目标**
- 连续 2 轮无新有价值信息 -> 停止

### 工具选择指引

| 场景 | 推荐工具 |
|------|---------|
| 一般搜索/迭代搜索 | `quick_search` — 轻量快速，适合多轮 |
| 需要抓取搜索结果中的详情链接 | `url_fetch` — 获取页面全文 |
| 用户说"深入研究/深度分析" | `web_search(intelligent=true)` — 一站式深度搜索 |
| 用户给了 URL 要求分析 | `url_fetch` 或 `web_search(url="...")` |

### 示例

用户问："伊朗和美国最近的军事冲突情况"

第 1 轮: quick_search(query="伊朗 美国 军事冲突 2026")
-> 10 条结果，多数自媒体来源
-> 评估：来源权威性不足

第 2 轮: quick_search(query="Iran US military conflict 2026 site:reuters.com OR site:bbc.com")
-> 路透社、BBC 报道
-> 评估：权威来源 OK，需抓取详情

第 3 轮: url_fetch(urls="https://reuters.com/article/...")
-> 完整报道内容
-> 评估：6 维度达标，组织回答
